- **응답 :**
    

Cloudflare Tunnel은 로컬 서버에 외부 포트를 열지 않고도, 안전하게 Outbound 커넥션을 맺어 외부에서 가입된 도메인으로 접속할 수 있게 해주는 가장 강력한 무료 서비스입니다.

설정 방법은 크게 두 가지(CLI 로컬 파일 직접 관리형 / Zero Trust 대시보드 관리형)가 있으나, **대시보드 관리형**이 복잡한 `config.yml` 파일을 작성할 필요가 없고 UI에서 직관적으로 관리할 수 있어 훨씬 권장됩니다.

전체 설정 단계를 아래 순서대로 진행해 보세요.

## Cloudflare Tunnel 설정 4단계

**1.도메인을 Cloudflare DNS에 등록하기:**사전 준비 단계.

터널을 사용하기 위해서는 본인 소유의 도메인이 Cloudflare에 등록되어 작동 중이어야 합니다.

1. Cloudflare 로그인 후 **[Add a Site]** 버튼을 통해 소유한 도메인을 추가합니다.
    
2. 도메인 구매 대행업체(가비아, GoDaddy 등)의 관리 페이지로 이동하여, 네임서버를 Cloudflare가 안내하는 네임서버 주소로 변경합니다.
    
3. 대시보드에서 도메인 활성화 처리가 완료될 때까지 대기합니다.
    

**2.Zero Trust 대시보드에서 Tunnel 생성:**웹 대시보드 설정.

전체 네트워크를 통제할 가상 터널 입구를 생성합니다.

1. Cloudflare 메인 대시보드 왼쪽 메뉴에서 **[Zero Trust]**로 이동합니다 (처음 진입 시 간단한 무료 플랜 가입 절차가 진행됩니다).
    
2. Zero Trust 메뉴의 **[Networks]** ➡️ **[Tunnels]**로 이동한 뒤, **[Create a tunnel]**을 클릭합니다.
    
3. Connector 타입을 기본값인 **Cloudflared**로 선택하고, 터널 식별 이름(예: `my-local-web`)을 입력한 뒤 저장합니다.
    

**3.로컬 서버에 cloudflared 커넥터 설치 및 실행:**로컬 디바이스 설정.

터널 구동을 위한 에이전트 데몬 프로그램을 설치하고 터널 토큰을 활성화합니다.

- 화면에 표시되는 운영체제(Windows, macOS, Linux 등) 중 본인의 환경을 선택합니다.
    
- 화면에 자동 생성된 **설치 및 서비스 실행 명령어**를 그대로 복사하여 로컬 서버 터널 터미널에 붙여넣고 실행합니다.
    

_예시 (Ubuntu/Debian Linux 환경일 경우):_

Bash

```
# cloudflared 패키지 다운로드 및 설치
curl -L --output cloudflared.deb https://github.com/cloudflare/cloudflared/releases/latest/download/cloudflared-linux-amd64.deb
sudo dpkg -i cloudflared.deb

# 서비스 등록 및 실행 (대시보드에서 제공한 토큰 포함 명령어 실행)
sudo cloudflared service install YOUR_UNIQUE_TOKEN_HERE
```

- 터미널에서 실행을 완료하면, Cloudflare 대시보드 화면 하단의 **Connectors** 상태가 **Active**로 바뀌며 연결을 감지합니다.
    

**4.공개 도메인과 로컬 포트 매핑 (Public Hostname):**최종 연결 테스트.

외부 도메인 주소로 접속했을 때, 어느 로컬 포트로 보내줄지 규칙을 정의합니다.

1. 터널 생성 완료 화면의 **Public Hostname** 탭을 클릭합니다.
    
2. **Subdomain**에 외부에서 접속할 주소(예: `test` 혹은 `api`)를 적고, **Domain** 드롭다운에서 등록한 내 도메인을 선택합니다.
    
3. **Service** 항목에 내부 로컬 주소 정보를 입력합니다.
    
    - **Type:** `HTTP` (혹은 상황에 맞게 선택)
        
    - **URL:** `localhost:8080` (실제 구동 중인 로컬 웹 서버 포트 입력)
        
4. 하단의 **[Save hostname]**을 클릭하여 저장합니다.
    

> **💡 확인하기**
> 
> 설정이 정상적으로 완료되었다면 웹 브라우저 주소창에 `[https://test.yourdomain.com](https://test.yourdomain.com)`을 입력했을 때 별도의 SSL 인증서 수동 설정 없이도 안전하게 HTTPS(L4-L7 프록시 지원) 보안 연결이 적용된 로컬 서버 화면을 확인할 수 있습니다.
