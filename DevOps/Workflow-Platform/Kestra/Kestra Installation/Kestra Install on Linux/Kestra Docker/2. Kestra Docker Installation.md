# 📚 Kestra 시작 가이드 (한국어)

> **목표**: Docker로 Kestra를 실행하고, 최초 사용자 계정을 만들고, 간단한 *Hello World* 플로우를 작성·실행하는 전체 과정을 단계별로 안내합니다.  
> 아래 내용은 **GitHub 레포 (`v1.0.5` 태그)** 와 **공식 Quickstart 문서** 를 종합했습니다.

---

## 1️⃣ 사전 요구사항

| 항목 | 권장 버전 |
|------|------------|
| **Docker** | 20.10+ (Docker Desktop 권장) |
| **OS** | Windows 10/11 (WSL2 권장) · macOS · Linux |
| **네트워크** | 포트 `8080` (UI)와 `Docker socket` 접근 권한 필요 |

> **Tip**: Windows 사용자는 *WSL2* 혹은 *PowerShell* 로 동일 명령을 실행할 수 있습니다.

---

## 2️⃣ Docker 로 Kestra 실행

### 2.1 기본 실행 (임시 DB, 데이터 비영구)

```bash
docker run --pull=always --rm -it -p 8080:8080 \
  --user=root \
  -v /var/run/docker.sock:/var/run/docker.sock \
  -v /tmp:/tmp \
  kestra/kestra:latest server local
```

- `-p 8080:8080` → UI 포트 매핑  
- `-v /var/run/docker.sock:/var/run/docker.sock` → Docker 안의 태스크가 Docker를 사용할 수 있게 함  
- `-v /tmp:/tmp` → 임시 파일 공유 (스크립트, 로그 등)

### 2.2 macOS 전용 옵션

macOS에서 `SVE` 관련 오류가 발생할 수 있으니 `JAVA_OPTS` 를 추가합니다.

```bash
docker run --pull=always --rm -it -p 8080:8080 \
  --user=root \
  -v /var/run/docker.sock:/var/run/docker.sock \
  -v /tmp:/tmp \
  -e JAVA_OPTS="-XX:UseSVE=0" \
  kestra/kestra:latest server local
```

### 2.3 Windows (PowerShell) 예시

```powershell
docker run --pull=always --rm -it -p 8080:8080 --user=root `
    -v "/var/run/docker.sock:/var/run/docker.sock" `
    -v "C:/Temp:/tmp" kestra/kestra:latest server local
```

> **⚠️ 주의**  
> - 위 명령은 **임시 H2 데이터베이스**를 사용하므로 컨테이너 재시작 시 데이터가 사라집니다.  
> - 영구 저장소가 필요하면 *Docker Compose* 를 사용하여 PostgreSQL 등 외부 DB와 연동합니다 (아래 **섹션 9** 참고).

---

## 3️⃣ 웹 UI 접속 및 최초 사용자 생성

1. 브라우저에서 `http://localhost:8080` 열기  
2. **첫 로그인 화면**이 뜨면 **관리자 계정**을 생성합니다.  
   - `username` : `admin` (예시)  
   - `password` : 원하는 비밀번호  

> **Tip**: `admin` 계정은 최초 로그인 후 **Settings → Users** 에서 추가 사용자와 권한을 관리할 수 있습니다.

---

## 4️⃣ API 인증 확인 (cURL 예시)

```bash
curl -u "admin:YOUR_PASSWORD" \
  -X POST \
  "http://localhost:8080/api/v1/executions/company.team/getting_started"
```

- 정상 응답(`200 OK`)이면 API 인증이 정상 동작함을 의미합니다.  
- `company.team` 은 네임스페이스와 플로우 ID 예시이며, 아직 존재하지 않아도 `401` 이 아닌 `404` 가 반환됩니다.

---

## 5️⃣ 첫 번째 플로우 만들기

### 5.1 UI 로 플로우 생성

1. 좌측 **Flows** 메뉴 → **Create** 버튼 클릭  
2. 아래 YAML을 복사·붙여넣기 → **Save**  

```yaml
id: getting_started
namespace: company.team

tasks:
  - id: hello_world
    type: io.kestra.plugin.core.log.Log
    message: "Hello, World!"
```

- `id` : 플로우 식별자  
- `namespace` : 조직/팀 구분 (예: `company.team`)  
- `type` : 사용 플러그인 (`io.kestra.plugin.core.log.Log`)  
- `message` : 로그에 출력될 문자열

### 5.2 플로우 실행 및 결과 확인

1. 플로우 리스트에서 방금 만든 플로우 → **Execute** 클릭  
2. 실행 내역이 **Executions** 탭에 표시됩니다.  
3. 실행 상세 화면 → **Logs** 탭에서 `Hello, World!` 메시지 확인.

## 6️⃣ 핵심 개념 정리

| 개념 | 설명 |
|------|------|
| **Flow** | 워크플로 전체 정의 (YAML) |
| **Task** | Flow 안의 개별 작업 (플러그인 기반) |
| **Namespace** | Flow를 논리적으로 묶는 단위 (팀·프로젝트) |
| **Trigger** | 자동 실행 조건 (스케줄, 이벤트 등) |
| **Plugin** | 외부 시스템·언어와 연동하는 모듈 (예: `log`, `http`, `script`) |
| **Execution** | Flow가 실제 실행된 인스턴스 |
| **Revision** | Flow 변경 이력을 저장, UI·Git 연계에 사용 |

## 7️⃣ 플러그인 활용 기본

- **코어 플러그인**은 기본 이미지에 포함 (`log`, `http`, `email` 등)  
- **외부 플러그인**을 설치하려면 `plugins` 폴더에 JAR 파일을 마운트하거나 Docker 이미지에 포함시킵니다.  
- 플러그인 리스트는 UI → **Plugins** 혹은 📚 **Docs → Plugins** 에서 확인 가능.

---

## 8️⃣ 영구 데이터베이스 연동 (Docker Compose)

임시 H2 대신 **PostgreSQL** 등 영구 DB를 사용하려면 아래 `docker-compose.yml` 예시를 참고하세요.

```yaml
version: "3.8"
services:
  postgres:
    image: postgres:15
    environment:
      POSTGRES_USER: kestra
      POSTGRES_PASSWORD: kestra
      POSTGRES_DB: kestra
    volumes:
      - pg-data:/var/lib/postgresql/data

  kestra:
    image: kestra/kestra:latest
    command: ["server", "local"]
    ports:
      - "8080:8080"
    environment:
      KESTRA_DATABASE_URL: jdbc:postgresql://postgres:5432/kestra
      KESTRA_DATABASE_USERNAME: kestra
      KESTRA_DATABASE_PASSWORD: kestra
    depends_on:
      - postgres
    volumes:
      - /var/run/docker.sock:/var/run/docker.sock
      - ./plugins:/opt/kestra/plugins   # (옵션) 커스텀 플러그인 폴더

volumes:
  pg-data:
```

```bash
docker compose up -d
```

- 이제 플로우와 실행 로그가 PostgreSQL에 영구 저장됩니다.  
- DB 설정은 `application.yml` 또는 환경변수(`KESTRA_DATABASE_*`) 로 조정 가능합니다.
