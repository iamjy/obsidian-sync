**사용 팁**

- 문서가 **DevOps**라면 파트 구성 예: ①문서 맥락 → ②흐름(배포 리드타임/파이프라인) → ③피드백(텔레메트리·알림·MTTR) → ④지속적 학습/실험(A/B, 가설) → ⑤보안/컴플라이언스 내재화 → ⑥조직/역할·체크리스트.
    
- 문서가 **Git**이라면 파트 구성 예: ①Git 정신/저장소 구조 → ②브랜치 전략(Git Flow/Trunk-Based 등) → ③커밋 메시지/규칙 → ④PR·코드리뷰 → ⑤머지/리베이스/충돌해결 → ⑥릴리즈/태그/체인지로그 → ⑦서브모듈/서브트리 → ⑧CI와 연동.
    

**검증 포인트(내부 체크)**

- 각 파트 끝에 미니퀴즈와 체크박스 액션이 있는지 확인하세요.
