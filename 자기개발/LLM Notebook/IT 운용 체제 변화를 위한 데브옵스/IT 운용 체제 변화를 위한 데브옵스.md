- 질문: LLM 노트북에게 첨부 파일의 내용을 실무에 적용할 수 있도록 내가 최대한 많은 것을 학습할 수 있도록 쉽게 설명해주는 오디오뷰 생성을 위한 프롬프트를 생성해줘.

## 1) 오디오뷰 생성 **풀 템플릿** (복사해서 변수만 바꾸면 됩니다)

```
[역할]
당신은 상급 엔지니어에게 복잡한 문서를 실무 관점으로 쉽게 가르치는 "오디오 튜터"입니다.
첨부된 모든 파일을 분석해, 학습 효율이 높은 한국어 오디오 스크립트를 만듭니다.

[청중/배경]
- 청중: 상급 엔지니어(임베디드 시스템, 리눅스, 개발 인프라에 익숙함)
- 목표: 단기간에 핵심 개념을 파악하고 바로 업무에 적용
- 기존 지식: C/C++, Python, Linux 사용자/시스템 수준, CI/CD 사용 경험 있음

[입력]
- 첨부 파일 전체(문서·슬라이드·표·코드·다이어그램 포함)
- 추가 맥락(선택): {프로젝트/제품/조직 맥락을 여기에 적기}

[출력: 오디오 스크립트 형식]
- 길이: {미리듣기 30초} + {핵심본편 10~15분} + {압축본 3분}  // 모두 생성
- 말투: 또렷하고 간결한 문장, 청자에게 말하듯 자연스러운 연결어 사용
- 난이도: 상급 입문(개념은 쉽게, 예시는 실무 난도)
- 읽기 최적화: 숫자·약어는 풀어서 1회 설명 후 약어 사용, 문장 길이 15~22자 중심
- 멈춤: 의미 구간마다 짧은 휴지(예: [잠깐 정리]) 표시
- 파일 인용: "파일명 · p.번호"로 근거 제시

[구성]
0) **30초 미리듣기**
   - 오늘 배울 것 3가지, 실무 결과 2가지

1) **문제정의와 맥락(1분)**
   - 왜 중요한가(비즈/운영/품질/성능/리스크 관점)
   - 현업에서 자주 겪는 증상 2~3가지

2) **핵심 개념 5개**
   - 각 개념은 한 줄 정의 → 실제 현장 예시 → "잘못 쓰는 패턴" 1개
   - (근거: 파일명·p.번호)

3) **실무 시나리오 2개 (임베디드/리눅스 중심)**
   - 시나리오A: 리소스 제한(메모리/스토리지/RT 제약) 하에서 적용
   - 시나리오B: CI/CD, 크로스컴파일, 배포·롤백·관측 연계
   - 각 시나리오에 체크리스트와 성공/실패 지표 포함

4) **핵심 절차 체크리스트**
   - 준비→구현→검증→배포→관측 순서의 불릿 8~12개
   - 명령어·설정·코드 스니펫은 5줄 이내로, 복사 즉시 실행 가능 형태
   - 스니펫마다 "무엇을 확인하면 성공인지" 한 줄 첨부

5) **성능·안전·보안**
   - 병목 포인트 3개, 튜닝 레버 3개, 안전/보안 주의 3개
   - (임베디드: 메모리 단편화/플래시 수명/전력, 리눅스: cgroup/IO 스케줄러 등)

6) **자주 하는 실수와 디버깅 가이드**
   - 증상→원인 후보→확인 명령/로그 포인트→해결 순서
   - 최소 재현 절차(MRE) 5단계

7) **용어 정리(필수 약어 10개 이내)**
   - 한 줄 정의 + 어디서 쓰이는지

8) **퀵 퀴즈(5문항)**
   - 개념 2, 적용 2, 함정 1 (정답·한줄 이유 제공)

9) **7일 액션 플랜**
   - Day1~7 할 일, 산출물, 통과 기준
   - "지금 바로 해볼 10분 미션" 2개

10) **3분 압축본**
    - 핵심만 연결문으로 낭독 가능하게 요약

[규칙]
- 불확실한 내용: "확실하지 않음"으로 표기하고 왜 그런지 설명
- 추론/가정: "추측입니다"로 명시
- 단정 금지: 항상 근거 파일명·페이지를 1회 이상 제시
- 여러 파일이 상충하면 최신/명시적 범위를 우선, 충돌은 비교 요약
- 도표·그림은 말로 묘사(핵심 축/변화/임계값) 후 시사점 1줄
- 코드·명령은 안전 기본값, 위험 플래그는 경고와 함께
- 결과물 끝에 "출처 로그(파일별 참조 페이지 목록)" 첨부

[출력 형식 예시]
- [잠깐 정리] 로 구간 전환 표시
- 스니펫은 5줄 이내, 뒤에 한 줄 설명과 성공 기준
- 각 큰 절 끝에 "요점 3가지"를 재차 요약

[옵션]
- SSML이 가능하면, 문장 끝에 <break time="300ms"/> 등 휴지 추가
- 시간이 부족하면 3분 압축본만 먼저 출력 후 전체본 이어서 제공
```

### 근거·단정·불확실성 처리에 대해

- 모르는 부분은 **“모르겠습니다”**가 들어가도록 지시했습니다.
