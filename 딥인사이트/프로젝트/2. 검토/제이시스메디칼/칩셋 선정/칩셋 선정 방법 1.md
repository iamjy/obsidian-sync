| **단계**                                                                  | **주요 활동 (Action Item)**                                           | **핵심 체크포인트 & 활용 도구**                                                                                                                                                                                                         |
| ----------------------------------------------------------------------- | ----------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **1. 요구사항 정의**                                                          | **제품 스펙 수치화**<br><br>  <br><br>- 모호한 표현 지양 (예: '빠르게' → '1GHz 이상') | • **성능:** MCU(제어) vs AP(OS 구동) vs NPU(AI)<br><br>  <br><br>• **메모리:** 내장 RAM/Flash 용량, 외장 DDR 필요 여부<br><br>  <br><br>• **I/O:** USB, Ethernet, CAN, MIPI, HDMI 등 필수 포트<br><br>  <br><br>• **환경:** 동작 온도 (상온 vs 산업용 -40~85°C) |
| **2. 후보군 탐색**                                                           | **벤더 및 칩셋 스크리닝**<br><br>  <br><br>- 스펙에 부합하는 칩 리스트업               | • **주요 벤더:** ST, NXP, TI, Microchip, Nordic 등<br><br>  <br><br>• **검색 도구:** DigiKey, Mouser, FindChips (필터 검색)<br><br>  <br><br>• **전략:** 주력 칩(Main)과 대안 칩(Alt) 2~3개 선정                                                      |
| **3. 생태계 확인**<br><br>  <br><br><span style="color:red">**(★중요)**</span> | **SW 개발 환경 검증**<br><br>  <br><br>- 하드웨어보다 SW 지원 여부가 더 중요함         | • **BSP 지원:** 공식 Linux/Android/RTOS SDK 제공 여부<br><br>  <br><br>• **문서화:** Datasheet, Ref. Manual, Errata(버그) 공개 여부<br><br>  <br><br>• **커뮤니티:** GitHub 예제 코드, E2E 포럼 활성화 정도                                                  |
| **4. 공급망 점검**                                                           | **양산 가능성(수급) 확인**<br><br>  <br><br>- 개발 후 부품을 못 구하는 사태 방지         | • **수명 주기:** Longevity Program (10~15년 공급 보증) 확인<br><br>  <br><br>• **납기(Lead Time):** 주문 후 입고까지 걸리는 주(Week) 수<br><br>  <br><br>• **가격:** SoC 단가 + PMIC + 메모리를 합친 **Total BOM** 계산                                           |
| **5. 실물 검증**                                                            | **개발 보드(EVB) 테스트**<br><br>  <br><br>- 데이터시트와 실제 성능의 괴리 확인         | • **POC:** 핵심 기능(통신 속도, AI 추론 등) 실제 구동<br><br>  <br><br>• **전력 측정:** 실제 동작 시 전류 소모량 측정<br><br>  <br><br>• **편의성:** 개발 툴(IDE), 컴파일러 설치 및 빌드 난이도 체감                                                                            |
| **6. 최종 확정**                                                            | **회로 설계 준비**<br><br>  <br><br>- 핀 충돌 및 기술 지원 체계 확인                | • **Pin Muxing:** 핀 설정 툴(CubeMX 등)로 기능 충돌 확인<br><br>  <br><br>• **기술 지원:** 문제 발생 시 대응 가능한 대리점(Distributor) 확보                                                                                                                |
