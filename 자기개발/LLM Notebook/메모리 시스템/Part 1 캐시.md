## 🧩 오디오 스크립트 상세 지시

### 오프닝 (30초)

- 오늘의 목표 3가지:
    
    1. 캐시의 존재 이유와 스크래치패드와의 차이 이해
        
    2. 로컬리티(특히 algorithmic locality)의 실전 의미 파악
        
    3. 조직/관리/일관성/계층을 **설계 트레이드오프**로 연결
        
- 이 강의 후 할 수 있는 것: 블록/셋/연관도 선택 이유를 설명, 프리페치/교체 정책의 부작용 예측, 멀티코어에서 write-back의 주의점 제시.
    

### 모듈 1. 캐시의 목적·투명성·스크래치패드 (2~3분)

- 캐시: 느린 저장장치 접근을 **locality**로 가속. 투명 동작(태그 사용) vs 스크래치패드(명시적 관리, 태그 無).
    
- 비유: “자동차 하이브리드의 배터리(빠른 소형 저장소)가 엔진(대용량 저장소)을 보완한다.”
    
- 3줄 요약 / 실무 팁.
    

### 모듈 2. 로컬리티 3종 (3분)

- **Temporal**: 곧 다시 쓴다 → 요구페치 후 보존.
    
- **Spatial**: 이웃도 쓴다 → 블록 크기·룩어헤드·프리페치.
    
- **Algorithmic**: 순서·패턴은 예측되지만 데이터가 흩어짐 → **런타임/컴파일러 프리페치, 버퍼 설계**.
    
- 연결 리스트 프리페치 예시(다음 노드 미리 청크인).
    
- 3줄 요약 / 실무 팁(“프레임 처리 루프에 ‘N 스텝 선행 프리페치’ 시도”).
    

### 모듈 3. 캐시 조직 (3분)

- **블록·태그·셋**: 주소 → [블록ID | 바이트오프셋], 태그는 “누가 들어왔는가”의 메타.
    
- 직접사상 ↔ 집합연관(N-way) ↔ 완전연관: **충돌/저장비용/탐색지연** 트레이드오프.
    
- 3줄 요약 / 실무 팁(“1MB 간격 충돌 주의—셋 매핑 확인”).
    

### 모듈 4. 컨텐츠 관리 (3분)

- 요구페치, **프리페치(하드/소프트)**, 교체(LRU/NMRU 등) 개념.
    
- 프리페치 과적재·오판 리스크와 힌트(히스토리 기반/스트라이드 탐지).
    
- 3줄 요약 / 실무 팁(“DMA 패턴을 캐시 정책과 align: 읽기 병목 시 ‘읽기 전용+프리페치’ 검토”).
    

### 모듈 5. 일관성 관리(개념 방향) (2~3분)

- **Write-through**: 단순·안전하지만 느림 → 쓰기 버퍼/결합 버퍼로 완충.
    
- **Write-back**: 대역폭 절약, 대신 더티 관리 및 멀티코어 상호작용 이슈.
    
- “캐시 일관성 vs 메모리 일관성(모델)” 구분 고지.
    
- 3줄 요약 / 실무 팁(“멀티스레드 공유 버퍼는 쓰기 정책·배치 정책을 함께 설계”).
    

### 모듈 6. 계층·포함/배제·파티셔닝 (2~3분)

- L1/L2/LLC와 **수직/수평 파티셔닝** 아이디어, **포함(inclusive) vs 배제(exclusive)** 개념적 차이.
    
- 전력/지연/적중률 최적화 직관 제공.
    
- 3줄 요약 / 실무 팁(“고주파 경로는 소형·근접, 백킹은 대용량·원격—데이터 핫셋을 식별하라”).
    

### 체크포인트 퀴즈 (약 1분)

    → 정답과 1문장 해설 제공.
