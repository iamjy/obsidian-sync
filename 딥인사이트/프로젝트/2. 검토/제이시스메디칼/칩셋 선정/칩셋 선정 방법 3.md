# 0. 킥오프(1~2일)

**목표**: 선정 범위와 의사결정 구조 확정  
**산출물**: R&R 매트릭스(기술/품질/구매/생산), 의사결정 기준 가중치(예: 성능30·공급망25·전력20·SW생태계15·단가10)

---

# 1. 요구사항·제약 수집(3~5일)

**무엇을 모으나**

- 기능: 센서/카메라 개수·인터페이스(MIPI CSI-2 lane 수, USB3, PCIe), 디스플레이, 코덱(ENC/DEC), 저장/통신(ETH/4G/5G/Wi-Fi)
    
- 실시간성: 하드/소프트 실시간 경로(예: 카메라→ISP/NPU→표시 지연 ≤ 80 ms)
    
- 성능: 필요한 TOPS/GMAC/s, CPU SPECint, GPU/ISP throughput, 메모리 대역폭
    
- 전력/열: 소비전력 한도, 히트싱크/팬 허용 여부, 폼팩터·보드 레이어 제약
    
- SW/BSP: 리눅스/RTOS, 메인라인 수준, 드라이버/멀티미디어 스택, 보안(secure boot/TPM/신뢰실행환경)
    
- 규제/안전: 의료/차량/산업(예: IEC 62304, ISO 26262 등 “적용 필요 여부”)
    
- 수명/공급: 최소 공급기간(예: 7년), 대체부품 정책, MOQ/리드타임
    
- 단가/TCO: SoC단가, 필수 라이선스(코덱·ISP·RTOS), 개발보드/툴/인증비
    

**산출물 템플릿(요구 스펙 요약표)**

|항목|값/범위|Must/Want|비고|
|---|---|---|---|
|카메라|2×MIPI CSI-2 (각 [4lane@2.5Gbps](mailto:4lane@2.5Gbps))|Must|동기화 필요|
|NPU|≥ 10 TOPS INT8|Want|모델: YOLOv8n-int8|
|메모리 BW|≥ 20 GB/s|Must|FHD@60 + NPU 병행|
|소비전력|≤ 7 W|Must|팬 없음|

---

# 2. 후보군 롱리스트(2~3일)

**방법**

- 카테고리로 넓게: General-purpose(AP), AI-Edge(NPU강), Industrial/Automotive(장기공급), MCU+AI 코프로
    
- 데이터 수집원: 벤더 자료서, 레퍼런스 디자인, 공개 BSP 브랜치/문서, 커뮤니티 포럼 활성도(이 단계는 “존재 여부 확인” 수준)
    

**산출물**: 10~20개 SoC의 스펙 표(요약)

---

# 3. 1차 정량 스크리닝(3~5일)

**합격기준 예시**

- I/O 일치(필수 인터페이스 100% 충족)
    
- 메모리 대역폭/코덱/ISP 스펙 하한선 통과
    
- 공급기간 ≥ 요구수명, 리드타임 ≤ 12주(예시)
    
- BSP: 리눅스 LTS 지원, 핵심 드라이버 상태(“메인라인/아웃오브트리”)
    
- 라이선스: 필요 코덱/ISP 라이선스 조달 가능
    

**퇴출 규칙(안전장치)**

- 필수 Must-Have 1개라도 미충족 → 탈락
    
- 벤더 로드맵 “단종 예정” 표기 → 탈락 보류(대체 SKU 확인 후 판단)
    

**산출물**: 4~6개 숏리스트

---

# 4. 성능·용량 계산(수식 기반 후보 비교, 2~4일)

**핵심 계산식 예시**

- **카메라 입력 대역폭** ≈ 해상도(픽셀) × FPS × 비트/픽셀 × 오버헤드(보통 1.2~1.5)
    
- **메모리 BW 요구** ≈ (카메라 입력 + NPU 중간 텐서 RW + 디스플레이/코덱 RW) 합
    
- **NPU 필요치** ≈ 모델 GOPS × FPS / 활용율(0.6~0.8)
    
- **열 설계**: SoC TDP ≤ 방열설계 용량(히트싱크/섀시 열저항)
    

**산출물**: 각 후보의 계산 시트(엑셀/마크다운), 병목(메모리/NPU/ISP) 표시

---

# 5. 숏리스트 POC/벤치마크(2~4주)

**실험 설계**

- **리얼 파이프라인** 기준 벤치마크: _Camera → ISP/NPU → Overlay → Encode → Display_
    
- **측정 항목**: E2E 지연(ms), 드롭률(%), 전력(W), 온도(℃), 스루풋(FPS), 부팅시간(s)
    
- **SW 품질**: BSP 안정성(장시간 soak test), 드라이버 completeness, 문서 품질, 커널 버전/LTS
    
- **개발자 경험**: 툴체인, 디버깅(JTAG/ETM), 프로파일러, 예제/SDK, 포럼 응답속도
    

**산출물 템플릿(POC 리포트)**

|항목|후보A|후보B|기준|
|---|--:|--:|--:|
|E2E 지연|68 ms|92 ms|≤ 80|
|소비전력|6.2 W|7.8 W|≤ 7.0|
|장시간 안정성(24h)|Pass|Drop 0.5%|Pass|
|BSP 이슈 수|3|9|적을수록 좋음|

---

# 6. 보안·안전·규제 검토(병행, 1~2주)

**체크리스트**

- Secure Boot/Key storage/TEE/암호가속
    
- OTA/롤백/이미지 서명 체계
    
- 취약점 공지·CVE 대응 프로세스
    
- 의료/자동차/산업용 요구 시: 안전수준 등급 지원, 장기공급(10~15년) 라인업, 벤더 품질시스템
    
- SBOM 지원, 오픈소스 라이선스 준수 체계
    

**산출물**: 규제 적합성 갭 분석표, 보완 계획

---

# 7. 공급망·수명·대체전략(1주)

**확인 포인트**

- 공급기간 공식 문서, 대체핀 호환 SKU, 멀티소싱 가능 여부
    
- 리드타임/Price-break, MOQ, 환율·관세 영향
    
- 벤더 기술지원 SLA(응답/버그픽스 리드타임), 레퍼런스 디자인 제공 범위
    

**산출물**: 공급위험 점수, 완화계획(세컨더리 BoM, 세컨드 SoC 경로)

---

# 8. 비용·TCO 분석(3~5일)

**구성**

- 단가(SoC, PMIC, 메모리, 전원/열/보드 레이어 변화)
    
- **NRE**(SDK/코덱 라이선스, 개발보드, 인증, 설계/레이아웃 리스핀)
    
- **운영비**(테스트·검증 인력, 유지보수, 보안패치, 장기부품 보관비)
    
- **일정 리스크 비용**(대체 설계 전환 가능성)
    

**산출물**: 3년/5년 TCO 비교 그래프, 민감도 분석(물량/환율 변화)

---

# 9. 의사결정 매트릭스(최종 1~2일)

**예시 가중치 적용 점수표**

|기준|가중치|후보A|후보B|
|---|--:|--:|--:|
|성능/지연|0.30|0.27|0.18|
|전력/열|0.20|0.16|0.12|
|SW/BSP|0.15|0.12|0.09|
|공급망|0.25|0.20|0.15|
|비용/TCO|0.10|0.07|0.08|
|**총점**|1.00|**0.82**|0.62|

**산출물**: 선택 사유서(정량 결과 + 주요 리스크와 대응)

---

# 10. 계약·초도양산 준비(동결 단계)

- **PN 동결**: SoC 리비전/스테핑 명시, 필수 문서(데이터시트, 하드웨어 가이드, 레이아웃 룰) 버전 고정
    
- **레퍼런스 디자인 활용**: 전원/DDR/고속신호 라우팅 룰 검토, SI/PI 시뮬레이션 계획
    
- **BSP 브랜치 전략**: LTS 커널 버전 고정, 보안패치 계획, 장기 유지보수 체계(백포트 정책)
    
- **양산 리스크 리허설**: 온도/전원 변동/EMC 프리-스캔, 부트/OTA 실패 복구 시나리오 드릴
    

---

## 실무 체크리스트(압축판)

-  Must-Have I/O 100% 충족
    
-  E2E 지연/전력 한도 통과(실측)
    
-  메모리 BW 여유 ≥ 20%
    
-  BSP 안정성(24~72h soak) 및 메인라인 진입 경로
    
-  Secure Boot/OTA/키관리 가능
    
-  장기공급 문서화 및 대체 SKU 경로
    
-  3~5년 TCO 우위
    
-  벤더 지원 응답성 검증(티켓 파일럿)
    

---

## 자주 겪는 함정(회피법)

- **데이터시트 수치만 보고 결정** → 실제 파이프라인 POC 필수
    
- **NPU TOPS 숫자만 비교** → 모델/연산자 지원, 메모리 병목 확인
    
- **BSP “지원” 문구 과신** → 드라이버 상태(메인라인/아웃오브트리), 예제 유무, 버그 대응 속도 확인
    
- **열 설계 과소평가** → 섀시·방열 부품을 포함한 시스템 열예산으로 검증
    
- **단가만 보고 선택** → 재설계/지연 비용 포함한 TCO로 비교

## 빠른 시작을 위한 산출물 템플릿(복붙용)

**A. 요구사항 요약(MoSCoW)**

```
Must: 2×CSI-2(4L@2.5Gbps), E2E≤80ms, 전력≤7W, 5년공급
Should: NPU≥10TOPS INT8, H.265 1080p60 ENC
Could: PCIe Gen3 x2, 2×Ethernet
Won’t: 4K60 디스플레이
```

**B. 벤치마크 스크립트 체크리스트**

- 카메라 캡처 → ISP 파이프 파라미터 고정(게인/노이즈)
    
- NPU 런타임(ONNX/TFLite) 동일 입력, INT8/FP16 모드 비교
    
- Overlay/Compositor 고정, VSync On
    
- 60분·24시간 soak, 로그/전력/온도 수집
    

**C. 의사결정 표준 사유서(요지)**

- “후보A는 E2E 지연 68 ms로 기준(≤80 ms) 충족, 메모리 BW 여유 25%, BSP 이슈 3건/패치 ETA 2주, 5년 공급 보증 문서 확보. 총점 0.82로 대안 대비 TCO·공급망·지연에서 우위.”

## 참고(확실하지 않음/일반 가이드)

- 각 벤더의 공개 자료서, 하드웨어 디자인 가이드, LTS 커널 문서, 커뮤니티 포럼 품질은 시점/제품에 따라 상이합니다(확실하지 않음). 실제 프로젝트에서는 반드시 최신 문서와 벤더 FAE 확인이 필요합니다.
