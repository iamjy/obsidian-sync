## Docker 설치
다음 예시는 n8n을 Docker Compose 없이 바로 `docker run` 명령으로 실행하는 기본적인 방법입니다.  
필요에 따라 환경변수와 볼륨을 조정해 주세요.

```bash
# Docker 이미지를 가져옵니다
docker pull n8nio/n8n

# 컨테이너를 실행합니다
docker run \
  -d \
  --name n8n \
  -p 5678:5678 \
  -v ~/.n8n:/home/node/.n8n \
  -e N8N_BASIC_AUTH_ACTIVE=true \
  -e N8N_BASIC_AUTH_USER=admin \
  -e N8N_BASIC_AUTH_PASSWORD=비밀번호 \
  n8nio/n8n
```

### 주요 옵션

| 옵션                              | 설명                           |
| ------------------------------- | ---------------------------- |
| `-p 5678:5678`                  | n8n UI가 실행되는 포트 5678을 노출합니다. |
| `-v ~/.n8n:/home/node/.n8n`     | 설정과 데이터를 영구 저장합니다.           |
| `-e N8N_BASIC_AUTH_ACTIVE=true` | 기본 인증을 활성화합니다.               |
| `-e N8N_BASIC_AUTH_USER`        | 인증 사용자명                      |
| `-e N8N_BASIC_AUTH_PASSWORD`    | 인증 비밀번호                      |

> **Tip:**  
> 파일 기반 저장이 아닌 MySQL/ PostgreSQL 등 데이터베이스를 사용하고 싶다면 추가 `-e DB_TYPE`, `-e DB_HOST`, `-e DB_PORT`, `-e DB_DATABASE`, `-e DB_USER`, `-e DB_PASSWORD` 를 설정해 주세요.

위 예시는 가장 단순한 형태이며, 실제 운영 환경에서는 Docker Compose 파일을 사용하는 것이 관리 및 확장에 유리합니다.
